# DreamAgent 홈페이지 — 컨셉 정의서 (제작 마스터 스펙)

> 제작·Claude Code 빌드의 **단일 기준 문서**. 슬로건의 상세 근거는 `슬로건 정의서`, 메시징·페르소나·IA 디테일은 `기획안` 참조. 이 문서는 *확정된 결정만* 한 장으로 모은다.

---

## 한 줄 정체성
도메인별 AI 에이전트를 만드는 회사. **전문가가 필요했던 일을, 이제 사용자가 직접 하게 만든다.**

## 포지셔닝
- **제품 주도** 마케팅 쇼케이스 사이트 (실제 로그인·제품 진입 없음, 데모는 "연출").
- **슬림 MVP**: 홈 + 제품 3종 + 데모/문의. 나머지는 2차.
- **DX-AX는 2차 트랙** — 정보 제공 + AX 자가진단(인터랙티브). 전면 아님.
- 경쟁(ProMR·CSO24)과 같은 *제품* 링에서 싸운다.

## 비전 & 슬로건 (확정)
- **비전:** AI는 사람을 빼는 게 아니라 더 강하게 만든다 — 우리는 그게 현장에서 작동하게 만든다.
- **슬로건:** "이제 전문가는, 당신입니다."
- **핵심:** 권한 이동 (편의성은 *수단*, 가치는 *권한이 사용자에게 돌아오는 것*).
- 상세 분해·금지사항 → `슬로건 정의서`.

## 핵심 차별 3축
통제 가능성(HITL) · 도메인 전문성(멀티에이전트) · 현장 연결성(DB/API).

## 공통 엔진 (모든 에이전트의 DNA)
대시보드(데이터정리·지표·분석) · graph RAG 지식층(옵시디언풍 지식그래프·회의록) · 문서생성(PDF·PPT) · HITL + todo workflow · 자동화.

## 라인업
- **모아드림** — 마케팅 (데이터분석 · 소재/캠페인 · 미디어 디렉터 · 리포트, 4에이전트 협업)
- **키워드림** — 제약 영업 (실적분석 · 스케줄 · 통계 자동입력 · 재고위험 경고 · 교체 제안)
- **마음드림** — 상담 *(※ 정신건강 민감 영역 — 카피·기능 노출 수위 별도 관리)*
- `+ 확장 예정` 슬롯

## 두 고객, 하나의 슬로건
- 슬로건은 **결과 축 하나**로 (시장 넓고 욕망 보편).
- 두 고객은 **구조로 분기**: 결과를 직접 내고 싶은 실무자·대표 ↔ 통제·리스크 1순위 규제산업 의사결정자.
- 히어로 헤드 = 결과, 서브 = 통제 한 스푼. 섹션·제품 페이지에서 각자에게 말한다.

## 홈 섹션 흐름 (Linear 벤치마크 기반)
1. **히어로** — 슬로건 + 대시보드+챗 목업 + 단일 CTA(데모 요청)
2. **신뢰** — 파트너·MOU·파일럿 (로고 약하면 정직하게)
3. **공통 엔진** — 대시보드·graph RAG·HITL+todo를 각각 *실제 제품 화면*과 함께 스택
4. **작동 방식 데모** — todo workflow가 펼쳐지고 "제안 → 승인"(HITL). 권한 이동을 *말하지 않고 보여줌*
5. **라인업 3종 + 두 고객 분기** — 일관 카드 / 결과 트랙 · 통제 트랙
6. **지식그래프 쇼케이스 + 데이터 연동** — 그래프 펼쳐지는 차별 비주얼
7. **AX 자가진단(2차 코너) + 유튜브 데모**
8. **최종 CTA**(슬로건 수미상관) + 푸터(슬림)

## 벤치마크 차용 원리
- 히어로 = **Linear** (제품을 히어로로, 추상 0)
- 라인업 = **Relevance AI** (에이전트를 "팀"으로 — 단 *대체* 아닌 *부린다* 프레임으로)
- 작업 과정 = **Devin** (트레이스 노출 = 신뢰)
- 전체 흐름 = **Linear** (증명 → 제품 보여주는 기능 섹션 스택 → 단일 CTA)
- **관통 원리:** 실제 제품/과정을 보여준다 · 추상 그래픽 0 · 위는 감정(슬로건) 아래는 논리(기능·연동) · 단일 행동으로 수렴.

## 디자인 방향 (원칙 확정 / 구체 토큰은 design.md)
- **따뜻함 × 정밀** — Linear의 정밀함 + 우리만의 온기. AI 슬롭(보라/파랑 그라데이션·blob·과도 glass) 금지.
- **실제 제품 화면이 주인공.** 추상 이미지로 제품 가리지 않기.
- 한글 우선 — **Pretendard** 기준.
- → 구체 색·폰트·radius·간격은 `design.md`에서 확정 (다음 단계).
